## 测试步骤

### 第一步：启动应用测试
1. 启动应用
2. 观察控制台输出，应该看到：
   ```
   [App] 开始应用初始化...
   [App] 数据诊断结果: ...
   [App] 初始化结果: ...
   [App] 数据验证结果: ...
   [App] 应用初始化完成
   ```

### 第二步：访问数据管理界面
1. 进入设置页面
2. 查找"数据管理"选项
3. 点击进入数据管理界面

### 第三步：测试数据诊断
1. 在数据管理界面中点击"诊断数据状态"
2. 观察数据统计信息的显示
3. 检查诊断结果是否正确显示

### 第四步：测试数据清理
1. 点击"自动修复问题"按钮
2. 观察操作结果反馈
3. 点击"完整数据清理"按钮
4. 确认清理操作的结果

### 第五步：测试危险操作
1. 点击"强制重新初始化"
2. 确认操作完成后数据状态
3. 测试"重置所有数据"功能（谨慎操作）

## 预期结果

### 正常启动
- 应用启动时自动执行数据诊断
- 如果发现问题，自动进行修复
- 只在首次启动时创建默认数据

### 数据管理界面
- 显示准确的数据统计
- 诊断功能正常工作
- 清理操作成功执行
- 用户反馈清晰明确

### 数据完整性
- 无重复分类和标签
- 无孤立的订单和说明书
- 所有关系正确维护
- 默认数据得到保护

## 故障排除

### 如果构建失败
1. 检查 Xcode 版本是否兼容
2. 清理构建缓存：Product → Clean Build Folder
3. 重新构建项目

### 如果功能异常
1. 检查控制台日志输出
2. 确认 Core Data 模型正确
3. 验证服务依赖注入

### 如果数据丢失
1. 检查是否意外触发了重置操作
2. 确认备份数据的存在
3. 重新初始化默认数据

## 性能监控

### 关键指标
- 启动时间：应用启动到完成初始化的时间
- 诊断时间：数据诊断操作的执行时间
- 清理时间：数据清理操作的执行时间
- 内存使用：操作过程中的内存消耗

### 日志监控
观察以下日志输出：
```
[DataDiagnostics] 开始数据诊断...
[DataCleanup] 开始完整数据清理...
[DataInit] 开始应用初始化...
[DataValidation] 执行数据验证...
```
