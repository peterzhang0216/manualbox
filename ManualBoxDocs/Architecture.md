# ManualBox 架构与产品规划文档

## 1. 技术栈选择

### 1.1 核心框架
- **SwiftUI**: 用于构建跨平台（iOS/macOS）用户界面
- **CoreData**: 用于数据持久化
- **Combine**: 用于响应式编程和数据流管理
- **Swift Concurrency**: 用于异步操作和并发处理
- **UserNotifications**: 用于本地通知管理
- **CoreImage**: 用于图像处理
- **Vision**: 用于 OCR 文字识别
- **CloudKit**: 用于 iCloud 数据同步

### 1.2 设计原则
- 优先使用 Apple 原生框架
- 避免引入第三方依赖
- 保持代码简洁和可维护性
- 确保跨平台兼容性

## 2. 项目架构

### 2.1 目录结构
```
ManualBox/
├── App/
│   ├── ManualBoxApp.swift        # 应用程序入口
│   └── AppDelegate.swift         # 应用程序代理
├── Features/                     # 功能模块
│   ├── Products/                 # 产品管理
│   │   ├── Views/
│   │   ├── ViewModels/
│   │   └── Models/
│   ├── Categories/              # 分类管理
│   ├── Tags/                    # 标签管理
│   └── Settings/                # 设置管理
├── Core/                        # 核心功能
│   ├── Models/                  # 数据模型
│   │   ├── Entities/           # CoreData 实体
│   │   └── DTOs/               # 数据传输对象
│   ├── Services/               # 服务层
│   │   ├── StorageService/     # 存储服务
│   │   ├── NotificationService/# 通知服务
│   │   └── SyncService/        # 同步服务
│   └── Utils/                  # 工具类
│       ├── Extensions/         # Swift 扩展
│       └── Helpers/            # 辅助工具
├── UI/                         # 通用 UI 组件
│   ├── Components/             # 可复用组件
│   └── Resources/              # 资源文件
└── Tests/                      # 测试
    ├── UnitTests/             # 单元测试
    └── UITests/               # UI 测试
```

### 2.2 架构模式
采用 MVVM（Model-View-ViewModel）架构，结合 SwiftUI 的特性：

1. **Model 层**
   - CoreData 实体定义
   - 业务逻辑和数据操作
   - 数据验证和转换

2. **View 层**
   - SwiftUI 视图组件
   - 用户界面布局
   - 用户交互处理

3. **ViewModel 层**
   - 视图状态管理
   - 业务逻辑处理
   - 数据绑定和转换
